# Semana 11 - QA y retroalimentación en pares + plan de mejora

## Enunciado para Moodle

En esta actividad realizarás un proceso de aseguramiento de calidad sobre un proyecto web con JavaScript. El objetivo es diagnosticar errores frecuentes, priorizarlos, aplicar correcciones y construir un plan de mejora a partir de pruebas y retroalimentación de pares.

## Tema

Errores frecuentes en JavaScript; depuración sistemática; control de flujos.

## Tipo de actividad

Taller (Workshop) o Foro (Forum), con coevaluación.

## Modalidad

Grupal o individual con coevaluación. El docente indicará si la revisión se hará por parejas, equipos de 2 a 4 estudiantes o de forma individual con revisión cruzada.

## Producto esperado

Cada estudiante o equipo debe entregar:

1. Carpeta del proyecto corregido.
2. Registro de bugs priorizado.
3. Correcciones aplicadas.
4. Plan de mejora.
5. Formato de coevaluación diligenciado.
6. Matriz de pruebas de regresión.
7. Evidencias de consola, pruebas y funcionamiento.

## Instrucciones generales

1. Abre el proyecto en Visual Studio Code.
2. Ejecuta `proyecto_a_revisar/index.html` en el navegador.
3. Abre la consola del navegador.
4. Identifica errores de JavaScript y problemas funcionales.
5. Registra cada bug en la plantilla correspondiente.
6. Prioriza los bugs encontrados.
7. Intercambia retroalimentación con un par o equipo.
8. Corrige los errores priorizados.
9. Ejecuta pruebas funcionales y pruebas de regresión.
10. Construye un plan de mejora con acciones pendientes y oportunidades de optimización.

## Criterio central

No se evalúa solamente que el código funcione. También se evalúa la calidad del diagnóstico, la claridad del registro de bugs, la justificación de la prioridad, la pertinencia de las correcciones, la calidad de la retroalimentación y la capacidad de proponer mejoras realistas.
