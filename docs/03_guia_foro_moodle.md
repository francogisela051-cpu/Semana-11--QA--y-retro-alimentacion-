# Guía para foro de retroalimentación en Moodle

## Propósito del foro

El foro permite publicar hallazgos, recibir retroalimentación y construir propuestas de mejora entre pares.

## Publicación inicial sugerida

Cada estudiante o equipo debe publicar una entrada con esta estructura:

```md
## Equipo o estudiante

## Proyecto revisado

## Bugs principales encontrados
1. Bug 1 - prioridad - breve explicación.
2. Bug 2 - prioridad - breve explicación.
3. Bug 3 - prioridad - breve explicación.

## Evidencia
Adjuntar capturas o describir ubicación de los archivos en la carpeta `evidencias/`.

## Correcciones aplicadas
Explicar qué se corrigió y cómo se verificó.

## Pregunta para mis compañeros
¿Qué otra mejora recomiendan para fortalecer la usabilidad o la validación del sistema?
```

## Respuesta a un compañero

Al comentar el trabajo de otra persona o equipo, usa esta estructura:

```md
## Fortalezas observadas

## Bugs o riesgos detectados

## Recomendación técnica

## Pregunta de mejora

## Valoración respetuosa
```

## Reglas de participación

- Sé específico: evita decir únicamente “está bien” o “está mal”.
- Describe pasos para reproducir el bug.
- Usa lenguaje respetuoso.
- Diferencia una falla crítica de una sugerencia estética.
- Propón mejoras posibles de implementar.
